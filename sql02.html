1. The Basics

Retrieve Data

Use * for all columns, or list specific ones.

SELECT * FROM table; SELECT name, salary FROM table;

Remove Duplicates

Returns only unique values in a column.

SELECT DISTINCT city FROM table;

Renaming (Aliases)

Rename a column in the output for clarity.

SELECT name AS EmployeeName FROM table;

2. Filtering (WHERE)

Basic Conditions

WHERE age > 18 WHERE country = 'USA' WHERE status != 'Active'

Logic Gates

Pro Tip: AND happens before OR. Use parentheses to group logic.

WHERE col1 = 5 AND col2 = 6 WHERE (A=1 AND B=2) OR C=3

Pattern Matching (LIKE)

  • % matches anything (0+ chars).
  • _ matches exactly 1 char.
WHERE name LIKE 'A%'

Handling NULLs

Never use = NULL. It won't work.

WHERE email IS NULL WHERE phone IS NOT NULL

3. Functions

String Magic

LEN(str) / CHAR_LENGTH(str) UPPER(str) / LOWER(str) TRIM(str) -- Remove spaces SUBSTRING(str, start, len)

Math Helpers

ABS(-5) -> 5 ROUND(1.234, 2) -> 1.23 FLOOR(1.9) -> 1

4. Grouping & Totals

Aggregate Functions

Condenses many rows into one.

COUNT(*) SUM(col) AVG(col) MAX(col)

The Golden Rule of GROUP BY

"If you SELECT a column that isn't inside a SUM/COUNT/AVG, you MUST put it in GROUP BY."

SELECT country, COUNT(*)
FROM users
GROUP BY country;

WHERE vs HAVING

  • WHERE: Filters *raw data* before grouping.
  • HAVING: Filters *results* after grouping.
GROUP BY country
HAVING COUNT(*) > 5
